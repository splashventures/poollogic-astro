## Decision Log
- Skipped font preload: @fontsource font files get hashed names per build; can't preload with static URLs. `font-display: swap` already handles flash-of-unstyled-text.
- Used `widths={[640, 1024, 1920]}` for hero; Astro won't upscale past source (1024px), so only 640w and 1024w are generated. Mobile gets 30kB instead of 76kB.
- Removed `preloadImage` prop entirely from BaseLayout (was misleading: preloaded `/images/pool/hero-pool.webp` but Astro outputs `/_astro/hero-pool.*.webp`, causing a wasted download).
- Also fixed ogUrl for homepage to include trailing slash.

## Surprises & Discoveries
- Hero source image is actually 1024x1024 (square), not 1920x1080 as the previous `width`/`height` attributes suggested
- `hero-pool-mobile.webp` (8.2KB) exists in src/assets but was never used
- LCP preload link was actively harmful: browser would download `/images/pool/hero-pool.webp` (unreachable path) AND `/_astro/hero-pool.*.webp` (actual image)
- Header and FAQ scripts broke after View Transitions navigation: all dropdowns, mobile menu, and FAQ accordion stopped working on any page navigated to via client-side routing
- Homepage canonical URL was missing trailing slash, creating a canonical mismatch with the `trailingSlash: 'always'` configuration
- `formatPhone()` was a no-op function that just returned its input unchanged
