# Plan: Google Reviews Cloud Function + Astro Component

## Assumptions
- Places API is enabled on cabana-cac GCP project
- API key stored in Secret Manager as 'google-places-api-key'
- gcloud CLI available at /tmp/google-cloud-sdk/bin/gcloud
- Site is static Astro deployed on Vercel; reviews fetched at build time
- Existing Testimonials component at src/components/ui/Testimonials.astro to be replaced

## Acceptance Criteria
1. Cloud Function source exists at functions/google-reviews/index.js + package.json
2. Cloud Function deployed to cabana-cac as 'google-reviews' (gen2, us-central1, HTTP, unauthenticated)
3. GET /?placeIds=ChIJ... returns JSON with reviews array and metadata object
4. Reviews are cached in Cloud Storage (cabana-cac bucket, reviews/ prefix, 24h TTL)
5. Only 5-star reviews with text are returned
6. Astro component at src/components/ui/GoogleReviews.astro renders a horizontal-scroll carousel
7. GoogleReviews replaces Testimonials in src/pages/index.astro
8. `npm run build` succeeds with the new component
9. PR created targeting main on splashventures/poollogic-astro

## Decision Log
- Used gcloud from /tmp/google-cloud-sdk/bin/gcloud with CLOUDSDK_AUTH_ACCESS_TOKEN since WSL gcloud had expired credentials; ADC tokens worked
- Cloud Function uses Places API (New) with X-Goog-FieldMask headers rather than legacy Places API
- Cache per place ID in Cloud Storage (reviews/{placeId}.json) for granular invalidation
- Carousel uses CSS scroll-snap on mobile, switches to 3-column grid on lg breakpoints
- Review text truncated at 200 chars with "Read more on Google" link to the specific review

## Surprises & Discoveries
- Places API (New) returns reviews under `reviews` array with `text.text` nested structure
- The API returns 5 reviews max per place (Google API limit)
- PoolLogic has 44 total reviews, 4.9 average rating
