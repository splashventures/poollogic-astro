## Decision Log
- Used @tailwindcss/vite instead of @astrojs/tailwind (which requires Tailwind v3). Tailwind v4 uses the Vite plugin directly.
- Duplicated categories data inside getStaticPaths for resources/category/[slug].astro because Astro hoists getStaticPaths and module-level constants aren't available.
- Used Vercel static adapter which outputs redirects to .vercel/output/config.json as 301 redirects.

## Surprises & Discoveries
- Vite emits warnings about type-only imports (City, Service, BlogPost, Testimonial) but these are harmless and don't affect the build.
- Astro's getStaticPaths runs in a separate scope from the rest of the frontmatter, so module-level variables defined before it aren't accessible within it.
- The 224 page count includes: 1 homepage + 68 [slug] pages + 129 blog posts + 12 topic hubs + 2 category pages + 1 resources index + 11 static pages (about, kyle-bowman, services, reviews, commercial, near-me, cost, emergency, sem, privacy, terms-of-use, html-sitemap) + robots.txt
