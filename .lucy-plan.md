# Plan: Mobile CRO/SEO audit and fix for PoolLogic Astro site

## Assumptions
- Site is live at poollogicsd.com, built with Astro 5.7 + Tailwind v4
- Mobile-first approach already in place but needs optimization
- Key pages: homepage (index.astro), dynamic service/city pages ([slug].astro)
- Sticky mobile CTA bar already exists at bottom of viewport
- Header has responsive mobile menu with accordion for dropdowns

## Acceptance Criteria
1. Hero section on mobile: CTA buttons visible within first viewport (~600px height) on 375px-wide viewport without requiring scroll
2. Comparison table is readable on mobile: feature text not cramped at 375px width
3. FAQ section padding is appropriate on mobile (no double-padding effect)
4. Section vertical padding is proportionate on mobile (reduced from desktop values)
5. Introduction section shows text before images on mobile (content-first for CRO)
6. Header top bar provides useful info even on small screens
7. No horizontal overflow on 320px-375px viewports
8. Site builds successfully with `npm run build` after all changes

## Approach
1. Reduce hero min-height and content density on mobile to keep CTA above fold
2. Make comparison table horizontally scrollable on small screens
3. Reduce FAQ container px padding on mobile
4. Add mobile-specific section padding reductions across all sections
5. Reorder introduction section grid so text comes first on mobile
6. Polish header mobile bar (show hours on small screens)
7. Review and fix any horizontal overflow issues
8. Build and verify

## Decision Log
- Hid hero value props (3 checkmarks) on mobile to reduce content density above fold. Trust signals (stars + bonded/insured) still visible. Value props are repeated in TrustStats section right below.
- Changed hero gradient from bg-gradient-to-r to bg-gradient-to-b on mobile. On portrait screens the left-to-right gradient leaves the right side too bright; top-to-bottom ensures text area is always dark.
- Used overflow-x-auto on comparison table instead of stacking rows. Stacking would lose the visual comparison effect; horizontal scroll preserves the side-by-side format while preventing text cramping.
- Reduced hero min-height from 600px to 480px on mobile. With header (~100px) + 480px hero, the CTA buttons should appear above fold on most mobile viewports (667px+ for iPhone SE).
- Showed hours text on mobile (was hidden at sm:inline). Hours are useful info and help fill the otherwise-empty left side of the top bar.

## Surprises & Discoveries
- The hero value props (all chemicals, CPO-certified, same technician) were adding ~60px of content before CTAs on mobile; hiding them gives CTAs room above fold
- FAQ had px-8 inner padding which, combined with the container's px-4, meant 48px of padding per side on mobile (leaving only ~279px of content width on a 375px screen)
- Introduction section images appeared before text on mobile since grid order defaults to source order; adding order classes was a simple content-first CRO win
- Comparison table's equal-thirds grid caused feature text to wrap heavily on mobile; the "Digital service reports with photos" text needed 3 lines
